# MetaFox 5.2.3

**Release Date: May 13th, 2026**

## What's New

### **Passkey Authentication (New)**

No more forgotten passwords! MetaFox 5.2.3 introduces **passkey support**, allowing users to sign in securely using biometrics or a device PIN — no password required. Whether on the web or mobile, the login experience is now faster, safer, and more modern.

#### **Highlighted Features**

- Users can register and sign in using a passkey (fingerprint, Face ID, or device PIN) instead of a traditional password.
- Available on both web and mobile platforms.
- Passkeys are managed per device, providing a seamless and highly secure authentication experience.

### **Subscriptions**

- **Trial Period Before Payment (Web)** — Users can now enjoy a free trial period before being charged for a subscription plan, making it easier to explore premium features before committing.

### **Core**

- **Ban User from the Front End** — Administrators can now ban users directly from the front-end interface without needing to navigate to AdminCP.
- **Share on Social (Mobile)** — Users on mobile can now share content directly to their external social networks from within the app.
- **Auto Login After Verification (Mobile)** — After completing the account verification step during registration, users are now automatically logged in — no extra steps needed.
- **Android 16 Readiness** — The mobile app has been upgraded to React Native 0.79, laying the groundwork for full compatibility with Android 16 (Target API 36).
- **Mobile App Improvements** — The mobile app now launches faster and loads the feed more quickly, delivering a snappier experience right from the moment users open the app.
- **In-App Background Upload (Mobile)** — Users can now continue browsing the app while their media uploads in the background, resulting in a smoother and uninterrupted experience.
- **AdminCP Improvements** - All data grid tables now support configurable rows per page, making it easier to manage large datasets.

### **Social Connect**

- **Social Registration Intermediate Step (Mobile)** — A new step has been added to the social registration flow, allowing users to review and agree to the Terms of Service and opt into notifications before their account is created.

### **Feed**

- Fixed an issue where the admin could not edit the privacy settings when editing another user's feed post.
- Fixed an issue where the Pin Post warning message was incorrectly shown when the pinned post limit had not been reached (Mobile).
- Improvements time to first view on the news feed, resulting in a faster and more responsive experience.
- UI/UX improvements.

### **Videos**

- Fixed an issue where the Android app could not play videos with 10-bit color depth.
- UI/UX improvements.

### **Marketplace**

- Fixed an issue where users were unable to create a free listing.

### **Invitations**

- Fixed an issue where a 404 page was shown when an invitee created an account via social login.

### **Events**

- Fixed an error that occurred when users sent mass emails for an event.

### **And,**

- There are many bug fixes and minor UI/UX improvements across the platform, including fixes for **Photos**, **Polls**, **Quizzes**, **Getting Started**, **ChatGPT Bot**, **Subscriptions**, **Groups**, and more.
